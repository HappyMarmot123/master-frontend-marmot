# Well-Architected Framework

## 개요

AWS Well-Architected Framework는 클라우드 아키텍처를 설계, 운영하는 데 도움이 되는 일련의 모범 사례와 설계 원칙을 제공합니다.

## 5개 기둥 (Pillars)

### 1. 운영 우수성 (Operational Excellence)

**목표**: 시스템을 실행하고 모니터링하여 비즈니스 가치를 제공하고 지속적으로 개선

**핵심 원칙**:

- 코드로 인프라 관리 (Infrastructure as Code)
- 작은 변경을 자주 수행
- 운영 절차를 자동화
- 실패에 대비하고 실패로부터 학습
- 운영 이벤트를 일상 업무로 전환

**주요 서비스**:

- CloudFormation, CDK
- CloudWatch
- Systems Manager
- CodePipeline, CodeDeploy

### 2. 보안 (Security)

**목표**: 정보, 시스템, 자산을 보호하면서 비즈니스 가치를 제공

**핵심 원칙**:

- 모든 계층에서 보안 적용
- 최소 권한 원칙
- 추적 및 감사 활성화
- 자동화된 보안 모범 사례
- 데이터 보호

**주요 서비스**:

- IAM, IAM Identity Center
- KMS, Secrets Manager
- Shield, WAF
- GuardDuty
- CloudTrail, Config

### 3. 안정성 (Reliability)

**목표**: 워크로드가 예상대로 작동하고 요구사항을 충족

**핵심 원칙**:

- 복구 절차 테스트
- 자동 복구
- 수평 확장
- 용량 관리
- 변경 자동화

**주요 서비스**:

- Auto Scaling
- ELB
- Route 53
- Multi-AZ 배포
- 백업 및 복구 솔루션

### 4. 성능 효율성 (Performance Efficiency)

**목표**: 컴퓨팅 리소스를 효율적으로 사용

**핵심 원칙**:

- 기술 선택 시 유연성 유지
- 최신 기술 활용
- 글로벌 아키텍처 고려
- 서버리스 아키텍처 활용
- 실험을 통한 개선

**주요 서비스**:

- CloudFront
- Auto Scaling
- ElastiCache
- 읽기 전용 복제본
- Lambda, Fargate

### 5. 비용 최적화 (Cost Optimization)

**목표**: 최저 비용으로 시스템을 실행

**핵심 원칙**:

- 사용한 만큼만 지불
- 사용하지 않는 리소스 제거
- 적절한 리소스 유형 및 크기 선택
- 데이터 전송 비용 최적화
- 비용 관리

**주요 서비스**:

- Cost Explorer, Budgets
- 예약형 인스턴스, Savings Plans
- 스팟 인스턴스
- S3 수명 주기 관리
- Auto Scaling

## 시험 포인트

1. **각 기둥별 핵심 원칙 이해**

   - 각 기둥의 목표와 원칙을 명확히 구분

2. **서비스 선택 기준**

   - 시나리오에 따라 어떤 기둥이 가장 중요한지 판단
   - 각 기둥을 지원하는 서비스 선택

3. **트레이드오프 이해**

   - 보안 vs 비용 최적화
   - 성능 vs 비용
   - 안정성 vs 복잡성

4. **모범 사례 적용**
   - 시나리오 기반 문제에서 Well-Architected 원칙 적용

## 실전 적용 예시

### 고가용성 웹 애플리케이션

- **보안**: IAM, 보안 그룹, WAF
- **안정성**: Multi-AZ, Auto Scaling, ELB
- **성능**: CloudFront, ElastiCache
- **비용**: 예약형 인스턴스, S3 수명 주기
- **운영**: CloudWatch, CloudFormation

### 서버리스 애플리케이션

- **보안**: IAM 역할, Secrets Manager
- **안정성**: Lambda 재시도, DLQ
- **성능**: Lambda 동시성, API Gateway 캐싱
- **비용**: Lambda는 사용한 만큼만 지불
- **운영**: CloudWatch Logs

## 관련 서비스

- **Well-Architected Tool**: 아키텍처 검토 도구
- 모든 AWS 서비스는 하나 이상의 기둥과 연관
