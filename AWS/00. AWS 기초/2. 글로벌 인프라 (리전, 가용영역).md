# 글로벌 인프라 (리전, 가용영역)

## 개요
AWS의 글로벌 인프라는 전 세계에 분산된 데이터 센터로 구성되어 있으며, 고가용성과 낮은 지연 시간을 제공합니다.

## 핵심 개념

### 리전 (Region)
- **정의**: 지리적으로 분리된 독립적인 데이터 센터 그룹
- **특징**:
  - 각 리전은 완전히 독립적 (다른 리전의 장애에 영향 없음)
  - 최소 2개 이상의 가용 영역으로 구성
  - 리전 간 데이터 전송 비용 발생
  - 리전별로 서비스 가용성과 가격이 다를 수 있음

### 가용 영역 (Availability Zone, AZ)
- **정의**: 리전 내 독립적인 데이터 센터
- **특징**:
  - 각 AZ는 독립적인 전원, 네트워크, 냉각 시스템 보유
  - AZ 간 낮은 지연 시간 (일반적으로 1-2ms)
  - AZ 간 데이터 복제 가능
  - 단일 AZ 장애 시 다른 AZ가 자동으로 처리

### 엣지 로케이션 (Edge Location)
- **정의**: CloudFront와 Route 53을 위한 글로벌 캐시 위치
- **용도**: 콘텐츠 전송 네트워크(CDN) 및 DNS 쿼리 처리
- **특징**: 리전보다 훨씬 많은 수의 엣지 로케이션 존재

## 사용 사례

### Multi-AZ 배포
- **목적**: 고가용성 확보
- **구현**: 여러 AZ에 리소스 배포
- **예시**: RDS Multi-AZ, ELB를 통한 여러 AZ의 EC2 인스턴스

### Multi-Region 배포
- **목적**: 재해 복구, 지리적 근접성, 규정 준수
- **구현**: 여러 리전에 리소스 배포
- **예시**: 글로벌 애플리케이션, 데이터 주권 요구사항

## 시험 포인트

1. **고가용성 설계**
   - 최소 2개 이상의 AZ 사용
   - 단일 장애 지점(SPOF) 제거

2. **비용 최적화**
   - 리전 간 데이터 전송 비용 고려
   - 리전별 가격 차이 확인

3. **지연 시간 최소화**
   - 사용자와 가까운 리전 선택
   - AZ 간 통신은 빠르지만, 리전 간 통신은 느림

4. **규정 준수**
   - 데이터 거버넌스 요구사항에 따른 리전 선택
   - 일부 리전은 특정 규정 준수 인증 보유

## 모범 사례

1. **프로덕션 워크로드**: 항상 Multi-AZ 배포
2. **재해 복구**: Multi-Region 전략 고려
3. **데이터 복제**: AZ 간 복제는 빠르고 저렴, 리전 간 복제는 느리고 비쌈
4. **로드 밸런싱**: 여러 AZ에 분산 배치

## 관련 서비스
- **Route 53**: 글로벌 DNS 서비스
- **CloudFront**: 글로벌 CDN
- **Global Accelerator**: 글로벌 네트워크 성능 향상
- **RDS Multi-AZ**: 자동 장애 조치
- **ELB**: 여러 AZ에 트래픽 분산
