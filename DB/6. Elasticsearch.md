## Elasticsearch

대용량 데이터를 빠르고 효율적으로 검색·분석할 수 있는 NoSQL 기반의 검색 엔진입니다. 일반적인 데이터베이스와 달리, 검색에 특화된 구조로 설계되어 텍스트 데이터를 처리할 때 더욱 뛰어난 성능을 제공합니다.

기본적으로 JSON 형식의 문서를 저장하고, 이를 인덱스 구조로 관리합니다. 사용자는 특정 키워드나 조건을 기반으로 문서를 신속하고 유연하게 검색할 수 있습니다.

Elasticsearch의 핵심은 내부적으로 사용하는 역색인(inverted index) 구조입니다. 이는 전통적인 데이터베이스가 행(row) 단위로 데이터를 관리하는 것과 달리, 단어 중심으로 색인을 생성하여 검색 속도를 극대화합니다. 예를 들어, "Elasticsearch는 빠르다"라는 문장을 저장하면, 각 단어가 색인에 등록되고 해당 단어가 포함된 문서와 연결됩니다. 이 덕분에 사용자가 "빠르다"라는 단어를 검색할 때 관련 문서를 매우 빠르게 찾아낼 수 있습니다.

데이터가 많아지더라도 여러 서버에 분산 저장할 수 있으며, HTTP 요청을 통해 데이터를 쉽게 송수신할 수 있습니다. 또한 로그 수집이나 실시간 데이터 분석에도 효과적으로 활용됩니다.

## 단점

트랜잭션(Transaction) 롤백 미지원: 엘라스틱서치는 분산 시스템의 성능 향상을 위해 트랜잭션과 롤백 기능을 지원하지 않습니다.

업데이트의 비효율성: 기존 문서를 수정하는 업데이트(Update) 명령이 들어오면, ES는 기존 문서를 실제로 삭제하고 새로운 문서 전체를 생성하는 방식으로 처리합니다.

조인(Join)의 복잡성: 전통적인 RDBMS와 같은 관계형 조인(Join)을 기본적으로 지원하지 않습니다. 대신 Nested Modeling이나 Parent-Child 같은 비효율적인 모델링 방식을 사용해야 하므로, 관계가 복잡할 수록 비용이 비싸집니다. 그리고 인덱시 작업이 리소스 소모가 심합니다.
