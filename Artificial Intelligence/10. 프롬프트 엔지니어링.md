# 프롬프트 엔지니어링 (Prompt Engineering) 상세 분석

프롬프트 엔지니어링은 대규모 언어 모델(LLM)의 능력을 최대화하기 위해 입력되는 명령어, 질문, 또는 컨텍스트(프롬프트)를 설계하고 최적화하는 기법입니다. 이는 모델의 성능을 결정하는 핵심 요소이며, 단순히 질문을 던지는 것을 넘어 모델의 사고 과정과 역할을 의도적으로 제어하는 기술입니다.

## 프롬프트 작성의 기본 원칙

성공적인 프롬프트는 명확성, 컨텍스트, 제약 조건, 예시라는 네 가지 요소를 충족해야 합니다.

| 원칙                       | 상세 설명                                                                                                                       | 예시 (지시 강화)                                                                                                                                              |
| :------------------------- | :------------------------------------------------------------------------------------------------------------------------------ | :------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **명확하고 구체적인 지시** | **모호성**을 제거하고, 모델이 수행해야 할 **작업(Task)**, **제약 조건(Constraints)**, **목표(Goal)**를 명확히 명시합니다.       | "이 개념을 설명해줘" → **"이 개념을 중학생이 이해할 수 있는 수준으로 비유를 들어 150자 이내로 설명해줘."**                                                    |
| **컨텍스트 및 역할 부여**  | 모델이 답변을 생성하는 데 필요한 **배경 지식**을 제공하거나, 모델에게 특정 **페르소나(Persona)**나 **역할(Role)**을 부여합니다. | "번역해줘" → **"당신은 20년 경력의 전문 번역가입니다. 다음 문장을 딱딱한 직역이 아닌, 문맥에 맞는 자연스러운 한국어로 번역해 주세요. 원문: [Original Text]"** |
| **출력 형식 명시**         | 결과물의 **구조**와 **포맷**을 지정합니다. (마크다운, JSON, XML, 표 등) 이는 후속 작업(Post-processing) 자동화에 필수적입니다.  | "리스트를 만들어줘" → **"결과를 다음 JSON 형식으로 출력해줘: `{ "title": "...", "items": ["...", "...", "..."] }`"**                                          |
| **예시 제공 (Few-shot)**   | 원하는 입력-출력 쌍의 예시를 제공하여 모델이 원하는 **스타일, 톤, 형식**을 모방하도록 유도합니다.                               | **예시:** `입력: 사과 → 출력: 과일. 입력: 자동차 → 출력: 운송수단. 입력: 컴퓨터 → 출력: ?`                                                                    |
| **사고 체인 유도 (CoT)**   | 모델에게 최종 답을 내기 전에 **추론 과정을 단계적으로 설명**하도록 요청하여, 복잡한 문제 해결 능력과 정확성을 향상시킵니다.     | **"최종 답을 도출하기 전에, 먼저 이 문제를 풀기 위한 논리적 사고 과정을 단계별로 설명해줘. 그 후에 최종 답을 제시해."**                                       |

## 주요 프롬프트 엔지니어링 기법 (Advanced Techniques)

LLM의 성능을 극대화하기 위해 연구되고 사용되는 심화 프롬프팅 기술들입니다.

#### A. 제로샷 / 퓨샷 프롬프팅 (Zero-shot / Few-shot Prompting)

- **Zero-shot:** 모델에게 예시 없이 지시만으로 작업을 수행하게 합니다. 가장 기본적인 프롬프팅 방식입니다.
- **Few-shot:** 1~3개의 입력-출력 예시를 프롬프트에 포함시켜 문맥 내 학습(In-Context Learning) 능력을 활성화하고 원하는 결과 포맷과 스타일을 유도합니다.

#### B. 사고의 사슬 프롬프팅 (Chain-of-Thought, CoT)

- **기법:** 프롬프트에 "단계별로 생각해보자(Let's think step by step)"와 같은 구문을 추가하여 모델이 추론 과정을 명시적으로 출력하도록 강제합니다.
- **효과:** 복잡한 산술, 논리, 상식 추론 문제에서 최종 정확도를 비약적으로 개선합니다. 이는 모델의 내부 추론 경로를 따라가게 함으로써 오류 발생 가능성을 낮춥니다.

#### C. 재귀적 프롬프팅 (Iterative Prompting)

- **기법:** 한 번의 완벽한 출력을 기대하기보다, 모델의 첫 번째 출력물을 피드백(Feedback)으로 활용하여 프롬프트를 수정하고 재요청하는 반복적인 과정입니다.
- **예시:** `(1차 프롬프트) → (모델 응답) → (2차 프롬프트: "위 답변의 3번 항목에 대해 전문가적인 비판 3가지를 추가해줘.") → (개선된 응답)`

#### D. 메타 프롬프팅 (Meta Prompting)

- **개념:** 프롬프트 세션의 가장 상위에 모델이 항상 준수해야 할 전역 규칙이나 구조적 지침을 정의합니다.
- **용도:** API 호출, 에이전트 개발 등 장기적인 상호작용에서 모델의 일관된 행동(예: 항상 친절한 톤 유지, 특정 API 함수만 사용)을 강제합니다.
