### 1. 각 화면에서 한 가지 행동에만 집중하게 한다

각 화면에서 한 가지 행동에만 집중하게 한다는 원칙은 특히 모바일 UI 디자인에서 중요합니다. 스마트폰 화면은 아주 작기 때문에, 사용자가 한 화면에서 무엇을 해야 할지 명확해야 합니다.

사용자가 UI를 사용하면서 망설이거나 혼란스러워해서는 안 됩니다. 더불어 특별한 경우를 제외하고는 사용자가 UI를 쓰기 위해 학습해야 하는 일이 있어서는 안 됩니다. 이를 위한 가장 좋은 방법이 바로 이 원칙입니다. 사용자가 앱에서 어떤 화면을 보고 있을 때, 그 화면에서 해야 하는 행동만 분명하게 알려주면 사용자는 매끄럽게 앱을 사용할 수 있습니다.

예를 들어, 회원가입 화면에서는 회원가입에만 집중하도록 하고, 상품 상세 화면에서는 상품 정보 확인과 구매에만 집중하도록 합니다. 한 화면에 너무 많은 선택지나 행동이 있으면 사용자는 무엇을 해야 할지 망설이게 되고, 이는 사용자 이탈로 이어질 수 있습니다.

물론 모든 화면에서 한 가지 행동에만 집중하게 하도록 할 수는 없습니다. 많은 콘텐츠가 종합선물세트처럼 모여있는 화면이 사용자에게 좋은 경험을 주는 상황도 분명히 있습니다. 예를 들어, 뉴스 피드나 쇼핑몰 메인 페이지처럼 다양한 콘텐츠를 탐색하는 화면에서는 여러 선택지를 제공하는 것이 오히려 좋을 수 있습니다.

**페르소나와 디자인 선택**

이 원칙을 적용할 때 중요한 것은 '페르소나(Persona)'를 고려하는 것입니다. 페르소나는 제품을 사용하는 대표적인 사용자 그룹의 특성, 목표, 행동 패턴을 가상의 인물로 만든 것입니다. 같은 원칙이라도 타겟 사용자의 페르소나에 따라 디자인 접근 방식이 달라질 수 있습니다.

구글과 네이버의 웹사이트 디자인 차이가 이를 잘 보여줍니다. 구글의 메인 페이지는 매우 심플합니다. 검색창이 중심에 있고, 상단에 햄버거 아이콘을 통해 자주 가는 웹사이트들에 접근할 수 있습니다. 이는 사용자가 '검색'이라는 한 가지 행동에 집중하도록 설계된 것입니다. 구글의 페르소나는 "빠르게 원하는 정보를 검색하고 싶은 사용자"입니다. 따라서 불필요한 요소를 제거하고 검색에만 집중할 수 있도록 디자인했습니다.

반면 네이버는 대시보드 형태로 디자인되어 있습니다. 메인 페이지에 뉴스, 쇼핑, 웹툰 등 사용자들의 트래픽이 많은 서비스들을 한눈에 볼 수 있도록 배치했습니다. 네이버의 페르소나는 "다양한 정보와 서비스를 한 곳에서 탐색하고 싶은 사용자"입니다. 따라서 여러 콘텐츠를 종합선물세트처럼 모아서 제공하는 것이 더 나은 경험을 제공합니다.

이처럼 같은 "각 화면에서 한 가지 행동에만 집중하게 한다"는 원칙이라도, 구글은 검색이라는 단일 행동에 집중하고, 네이버는 탐색이라는 행동에 집중하되 여러 콘텐츠를 한 화면에 보여주는 방식으로 구현했습니다. 이는 각 서비스의 페르소나와 사용 목적이 다르기 때문입니다. 타겟 사용자의 페르소나를 명확히 정의하고, 그에 맞는 디자인 접근 방식을 선택해야 합니다.

하지만 생소한 서비스이거나 사용자에게 꼭 전달해야 하는 특별한 가치에 집중하는 서비스에서는 이 원칙을 철저하게 지키는 것이 최고의 경험을 줄 수 있습니다. 회원가입, 로그인, 결제 등 중요한 전환 지점에서는 각 화면에서 한 가지 행동에만 집중하도록 디자인하는 것이 효과적입니다.

### 2. 가시성 (Visibility)

가시성은 사용자가 제품의 기능과 상태를 쉽게 파악할 수 있도록 하는 원칙입니다. 사용자가 무엇을 할 수 있는지, 현재 어떤 상태인지가 명확하게 보여져야 합니다.

문을 열 때를 생각해보세요. 좋은 디자인은 문을 밀어야 하는지 당겨야 하는지가 명확하게 보여집니다. 예를 들어, 손잡이가 있는 문은 보통 당겨서 열고, 손잡이가 없는 평면 문은 밀어서 엽니다. 하지만 손잡이가 있는데도 밀어야 하는 문이 있다면, 사용자는 혼란스러워합니다.

웹사이트나 앱에서 버튼은 클릭 가능하다는 것이 명확하게 보여져야 합니다. 버튼이 회색으로 비활성화되어 있으면 클릭할 수 없다는 것을 알 수 있고, 마우스를 올렸을 때 색상이 변하거나 커서가 손가락 모양으로 바뀌면 클릭 가능하다는 것을 알 수 있습니다. 또한 현재 어떤 페이지에 있는지, 어떤 메뉴가 선택되어 있는지도 명확하게 표시되어야 합니다.

가시성이 부족하면 사용자는 제품의 기능을 발견하지 못하거나, 현재 상태를 파악하지 못해 혼란스러워합니다. 스마트폰의 설정 메뉴가 너무 깊숙이 숨겨져 있으면 차지 못할 수 있는 것처럼요.

### 3. 피드백 (Feedback)

피드백은 사용자의 행동에 대해 시스템이 적절한 반응을 보여주는 원칙입니다. 사용자가 어떤 행동을 했을 때, 그 행동이 제대로 전달되었는지, 어떤 결과가 발생했는지를 명확하게 알려줘야 합니다.

웹사이트에서 파일을 업로드할 때, 업로드 진행 상황을 프로그레스 바나 퍼센트로 보여주면 사용자는 파일이 업로드되고 있다는 것을 알 수 있습니다. 버튼을 클릭했을 때 버튼이 눌리는 애니메이션이나 색상 변화가 있으면 클릭이 제대로 되었다는 것을 알 수 있습니다. 폼을 제출했을 때 체크마크가 나타나면 제출이 완료되었다는 것을 알 수 있습니다.

**마이크로 인터랙션 (Micro Interactions)**

피드백 원칙이 가장 잘 적용되는 디자인의 꽃은 '마이크로 인터랙션'입니다. 마이크로 인터랙션은 사용자에게 섬세하게 반응하여, 사용자가 도달해야 하는 목표까지 매끄럽게 가도록 돕는 디자인입니다. 사용자가 무언가를 조작할 때 실시간으로 제공하여 사용자가 막힘없이 제품을 이용할 수 있게 합니다.

회원가입 과정에서 비밀번호를 입력하는 상황을 예로 들어보겠습니다. 사용자가 비밀번호를 입력하는 순간부터 실시간으로 피드백을 제공합니다다. 만약 사용자가 비밀번호를 모두 입력한 후 완료 버튼을 눌렀을 때, 유효성 검사에 실패하면서 입력값이 리셋된다면 사용자는 회원가입에서 이탈할 확률이 높아집니다.

좋은 마이크로 인터랙션은 다음과 같이 작동합니다:

- 사용자가 비밀번호를 1글자 이상 입력하는 순간부터 피드백을 실시간으로 제공합니다.
- 비밀번호가 조건을 충족했다면, 긍정의 은유적 색상인 초록색 글씨로 피드백을 제공합니다.
- 비밀번호가 모든 조건을 충족했다면 "계속하기" 버튼이 활성화되어 다음 단계로 진행할 수 있음을 알려줍니다.

이처럼 마이크로 인터랙션은 사용자가 입력하는 순간부터 실시간으로 상태를 알려주고, 다음 단계로 진행할 수 있는지 명확하게 표시합니다. 특히 버튼의 활성화/비활성화 상태를 명확히 구분하는 것이 중요합니다. 다음 단계로 진행할 수 없는데 버튼이 활성화되어 있으면, 사용자가 클릭할 수 있다고 착각하고 눌렀다가 경고 팝업을 보거나 반응이 없는 상황을 맞닥뜨리게 됩니다.

### 4. 제약 (Constraints)

제약은 사용자가 잘못된 행동을 하지 못하도록 제한하는 원칙입니다.제약에 따라 다양한 방법으로 사용자의 행동을 제한할 수 있습니다. 예를 들어, USB 케이블은 한 방향으로만 꽂을 수 있도록 설계되어 있습니다. 이것이 물리적 제약입니다. 만약 USB를 잘못된 방향으로 꽂으려고 하면 들어가지 않습니다.

웹사이트에서 필수 입력 필드를 채우지 않으면 제출 버튼이 비활성화되어 있거나, 제출을 시도하면 오류 메시지가 나타납니다. 이것이 논리적 제약입니다. 또한, 날짜 선택기에서 과거 날짜는 선택할 수 없도록 제한하거나, 숫자 입력 필드에서 숫자만 입력할 수 있도록 제한하는 것도 제약의 예입니다.

**제약의 종류:**

- **물리적 제약**: 물리적으로 불가능한 행동을 막는 것 (예: USB 케이블의 방향)
- **논리적 제약**: 논리적으로 다음 단계로만 진행할 수 있도록 하는 것 (예: 단계별 폼)
- **문화적 제약**: 문화적 관습에 따라 자연스럽게 이해되는 것 (예: 빨간색은 경고, 초록색은 안전)

제약이 적절하게 적용되면 사용자는 실수를 하기 어렵고, 올바른 방법으로 제품을 사용하게 됩니다. 하지만 제약이 너무 많으면 사용자가 원하는 행동을 하지 못해 불편할 수 있으므로, 적절한 균형이 필요합니다.

### 5. 매핑 (Mapping)

매핑은 제어 장치와 그 결과 사이의 관계가 직관적으로 이해될 수 있도록 하는 원칙입니다. 사용자가 어떤 조작을 하면 어떤 결과가 나올지 예측할 수 있어야 합니다.

자동차의 방향 지시등 레버를 위로 올리면 오른쪽 방향등이 켜지고, 아래로 내리면 왼쪽 방향등이 켜집니다. 이것은 레버의 움직임 방향과 방향등의 방향이 일치하는 좋은 매핑입니다. 하지만 만약 반대로 되어 있다면, 사용자는 혼란스러워할 것입니다.

웹사이트에서 스크롤바를 오른쪽으로 드래그하면 페이지가 아래로 스크롤되고, 왼쪽으로 드래그하면 위로 스크롤됩니다. 이것은 공간적 매핑의 예입니다. 또한, 볼륨 조절 슬라이더에서 오른쪽으로 이동하면 소리가 커지고, 왼쪽으로 이동하면 소리가 작아지는 것도 직관적인 매핑입니다.

매핑이 직관적이면 사용자는 제품을 사용하기 전에 어떻게 작동할지 예측할 수 있습니다. 하지만 매핑이 직관적이지 않으면 사용자는 학습해야 하고, 실수를 하기 쉽습니다.

### 6. 일관성 (Consistency)

일관성은 비슷한 상황에서 비슷한 방식으로 작동하도록 하는 원칙입니다. 사용자가 한 곳에서 배운 것을 다른 곳에서도 적용할 수 있어야 합니다.

대부분의 자동차에서 브레이크 페달은 왼쪽에, 가속 페달은 오른쪽에 있습니다. 만약 어떤 자동차는 반대로 되어 있다면, 운전자는 큰 사고를 낼 수 있습니다. 또한, 대부분의 문에서 손잡이를 시계 방향으로 돌리면 열리고, 반시계 방향으로 돌리면 잠깁니다.

웹사이트나 앱에서 "저장" 버튼은 항상 같은 위치에 있고, 같은 모양과 색상을 가져야 합니다. 또한, "취소" 버튼은 항상 왼쪽에, "확인" 버튼은 항상 오른쪽에 배치하는 것이 일관성 있는 디자인입니다. 메뉴 구조도 일관되게 유지되어야 합니다. 예를 들어, "설정" 메뉴는 항상 오른쪽 상단이나 하단에 위치하는 것이 좋습니다.

**일관성의 종류:**

- **내부 일관성**: 같은 제품 내에서 일관된 디자인 (예: 같은 앱의 모든 화면에서 버튼 스타일이 동일)
- **외부 일관성**: 다른 제품과의 일관성 (예: 모든 웹사이트에서 로고 클릭 시 홈으로 이동)
- **기능적 일관성**: 같은 기능은 항상 같은 방식으로 작동 (예: 모든 삭제 버튼은 빨간색)

### 7. 개념 모델 (Conceptual Model)

개념 모델은 사용자가 제품이 어떻게 작동하는지에 대한 정신적 모델입니다. 디자이너는 사용자가 제품을 이해할 수 있도록 명확하고 일관된 개념 모델을 제공해야 합니다.

사용자는 냉장고의 온도 조절 다이얼을 돌리면 냉장고 내부 온도가 변한다고 생각합니다. 하지만 실제로는 냉장고의 압축기 작동 시간을 조절하는 것일 수 있습니다. 사용자의 개념 모델과 실제 작동 방식이 다르면, 사용자는 냉장고를 제대로 사용하지 못할 수 있습니다.

사용자는 파일을 "삭제"하면 휴지통으로 이동하고, 휴지통을 비우면 영구적으로 삭제된다고 생각합니다. 이것이 사용자의 개념 모델입니다. 만약 파일을 삭제했는데 바로 영구적으로 삭제된다면, 사용자의 개념 모델과 다르므로 혼란스러워합니다. 또한, 사용자는 "저장" 버튼을 누르면 파일이 저장된다고 생각하지만, 일부 앱에서는 자동 저장 기능이 있어 "저장" 버튼이 없을 수도 있습니다. 이 경우 사용자의 개념 모델과 다르므로 혼란스러울 수 있습니다.

사용자가 제품을 이해하고 사용하려면 명확한 개념 모델이 필요합니다. 디자이너는 사용자의 개념 모델을 고려하여 디자인해야 하고, 필요하다면 사용자에게 개념 모델을 교육해야 합니다. 하지만 가능하면 사용자의 직관적인 개념 모델에 맞춰 디자인하는 것이 좋습니다.

### 8. 발견 가능성 (Discoverability)

발견 가능성은 사용자가 제품의 기능을 쉽게 발견할 수 있도록 하는 원칙입니다. 사용자가 무엇을 할 수 있는지, 어떻게 해야 하는지를 쉽게 알 수 있어야 합니다.

웹사이트의 네비게이션 메뉴는 사용자가 쉽게 발견할 수 있는 위치(보통 상단이나 좌측)에 배치되어야 합니다. 또한, 중요한 기능은 눈에 띄는 위치에 배치하거나, 사용자가 처음 방문했을 때 튜토리얼이나 도움말을 제공하여 기능을 발견할 수 있도록 해야 합니다. 버튼이나 링크는 클릭 가능하다는 것이 명확하게 보여져야 하고, 호버 효과나 툴팁을 통해 추가 정보를 제공할 수 있습니다.

기능이 숨겨져 있으면 사용자는 그 기능을 사용하지 못합니다. 이는 제품의 가치를 낮추고, 사용자 경험을 저하시킵니다.

### 9. 어포던스 (Affordance)

어포던스는 객체가 어떤 행동을 할 수 있게 해주는 속성입니다. 사용자가 객체를 보고 어떤 행동을 할 수 있는지 직관적으로 알 수 있어야 합니다.

의자는 앉을 수 있다는 어포던스를 가지고 있습니다. 의자의 모양과 크기를 보면 앉을 수 있다는 것을 알 수 있습니다. 하지만 벽은 앉을 수 없다는 것을 알 수 있습니다. 또한, 손잡이가 있는 문은 당기거나 밀 수 있다는 어포던스를 가지고 있습니다.

버튼의 모양(둥근 모서리, 그림자 효과 등)을 보면 클릭할 수 있다는 것을 알 수 있습니다. 입력 필드의 모양(테두리, 플레이스홀더 등)을 보면 텍스트를 입력할 수 있다는 것을 알 수 있습니다.

디자인 원칙들은 서로 독립적으로 작동하는 것이 아니라, 함께 작동하여 좋은 경험을 만들어냅니다. 버튼은 클릭할 수 있다는 어포던스를 가지고 있어야 하고, 클릭했을 때 피드백을 제공해야 하며, 일관된 위치와 스타일을 가져야 합니다.
