# Symbolic Link (심볼릭 링크)

심볼릭 링크(Symbolic Link)는 파일이나 디렉토리를 가리키는 특별한 종류의 파일입니다. 원본 파일이나 디렉토리의 경로를 저장하는 "별칭" 또는 "참조"라고 생각할 수 있습니다. Windows에서는 "바로가기"와 유사하지만, 운영체제 레벨에서 작동하며 프로그램이 이를 실제 파일처럼 인식할 수 있다는 점에서 다릅니다. 링크를 통해 파일에 접근하면, 운영체제가 자동으로 원본 파일로 연결해줍니다.

**예시**:

- 원본 파일: `/home/user/documents/report.pdf`
- 심볼릭 링크: `/home/user/desktop/report.pdf` (심볼릭 링크)
- 링크를 열면 → 실제로는 `/home/user/documents/report.pdf`가 열림

### 하드 링크와의 차이

**하드 링크**:

- 원본 파일과 동일한 데이터를 가리킴
- 원본 파일이 삭제되어도 하드 링크는 여전히 작동
- 같은 파일 시스템 내에서만 생성 가능
- 디렉토리에 대해 생성 불가

**심볼릭 링크**:

- 원본 파일의 경로를 저장
- 원본 파일이 삭제되면 링크가 깨짐 (broken link)
- 다른 파일 시스템, 네트워크 드라이브에도 생성 가능
- 디렉토리에 대해서도 생성 가능
- 더 유연하고 널리 사용됨

여러 프로젝트에서 공통 라이브러리를 사용할 때, 각 프로젝트에 라이브러리를 복사하는 대신 심볼릭 링크를 사용하면 디스크 공간을 절약하고 버전 관리가 쉬워집니다.

프로그램이 `/usr/bin/python`을 기대하지만, 실제로는 `/usr/bin/python3.9`가 설치되어 있을 때, 심볼릭 링크를 통해 연결할 수 있습니다.
