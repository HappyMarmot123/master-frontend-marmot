# Tailwind

Tailwind CSS는 유틸리티 우선(Utility-First) 방식의 CSS 프레임워크입니다. 성능을 저하시키지 않고 빠른 개발과과 편의성에 초점을 맞춥니다.

flex, pt-4(padding-top: 1rem), text-center(text-align: center) 등과 같이 단일 목적을 가진 수많은 작은 CSS 클래스(유틸리티 클래스)를 미리 정의하고 제공합니다. 개발자는 이 유틸리티 클래스들을 HTML 요소의 class 속성에 나열하여 디자인을 구성합니다.

Tailwind의 유틸리티 클래스는 매우 낮은 명시도(Low Specificity)를 갖도록 설계됩니다. 대부분의 유틸리티 클래스는 단일 클래스 셀렉터(.pt-4)로 정의됩니다.

이는 개발자가 직접 작성한 커스텀 CSS나 특정 컴포넌트 CSS가 유틸리티 클래스보다 높은 명시도를 갖도록 보장하여, 스타일 충돌을 최소화하고 필요할 때 오버라이드(Override)하기 쉽게 만듭니다.

초기에는 모든 유틸리티 클래스가 포함된 큰 CSS 파일이 제공되었으나, 현재는 JIT(Just-In-Time) 모드 또는 PostCSS 플러그인을 통해 빌드됩니다. 프로젝트의 HTML, JS, Vue, React 등 소스 코드를 스캔하여 실제로 사용된 유틸리티 클래스만 추출하고, 해당 클래스에 대응하는 코드를 최종 CSS 파일에 생성합니다.

사용하지 않는 CSS 코드는 최종 빌드 파일에 포함되지 않아 최적화된 작은 CSS 파일을 얻을 수 있습니다.

유틸리티 클래스는 자주 재사용되므로, 최종 CSS 파일이 변경될 일이 적습니다. 이는 브라우저 캐싱 효율을 높여줍니다.

Tailwind 유틸리티 클래스는 단일 클래스 셀렉터로 정의됩니다. 즉, 복합 셀렉터가 없다는 의미로 성능상 이점을 가질 수 있습니다.
